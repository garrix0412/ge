## 1. 项目概述

### 1.1 项目背景

加密货币市场近年来发展迅速，已成为全球金融市场的重要组成部分。与传统股票市场相比，加密货币市场具有以下独特特征：全天候交易（24/7）、高波动性、数据完全公开透明，以及更易受到舆情和市场操纵的影响。这些特征使其成为应用深度学习技术进行价格预测和异常检测的理想研究对象。

本项目旨在构建一个多模型对比的加密货币价格预测系统，并辅以市场异常检测功能，最终以可交互的 Web Dashboard 形式交付，为投资者和研究者提供一个实用的市场分析工具。

### 1.2 项目目标

- **核心目标：** 基于多种深度学习模型（LSTM、GRU、Transformer）对 BTC 和 ETH 进行价格预测，并与传统统计方法（ARIMA）和机器学习方法（XGBoost）进行系统性对比。
- **辅助目标：** 利用 LSTM-Autoencoder 构建市场异常检测模块，识别闪崩、异常交易量突增等异常事件。
- **系统交付：** 搭建基于 Streamlit 的可交互式 Web Dashboard，集成实时数据展示、价格预测、异常警报等功能。

### 1.3 跨学科视角

本项目融合了计算机科学（深度学习模型设计与实现）、金融学（技术分析指标与市场行为分析）以及数据科学（特征工程与数据可视化）。报告中还将讨论异常检测对散户投资者保护的意义以及加密货币市场监管的相关伦理问题。

## 2. 研究对象与数据

### 2.1 目标加密货币

| 币种 | 选择理由 |
|------|----------|
| BTC (Bitcoin) | 市值最大，历史数据最长（2009年至今），流动性最好，研究文献最丰富 |
| ETH (Ethereum) | 市值第二，智能合约生态系统代表，与 BTC 存在差异化的价格模式 |

在异常检测部分，将引入 1–2 个小市值币种作为案例分析对象，以验证异常检测模块对市场操纵行为（如 pump-and-dump）的识别能力。

### 2.2 数据来源

| 数据类型 | 来源 | 说明 |
|----------|------|------|
| 历史价格数据（OHLCV） | Binance API / CoinGecko API | 开盘价、最高价、最低价、收盘价、交易量 |
| 实时价格数据 | Binance WebSocket API | 用于 Dashboard 实时展示 |
| 链上数据（可选） | Glassnode / IntoTheBlock | 活跃地址数、交易所净流入流出、持币分布等 |
| 舆情数据（可选） | Twitter API / Reddit API | 用于情感分析辅助预测 |

**数据获取工具：** 使用 Python 的 `ccxt` 统一交易所接口库进行数据采集。

### 2.3 数据时间范围

- **训练数据：** 2020年1月 – 2025年6月（涵盖牛市、熊市、震荡市多种市场状态）
- **验证数据：** 2025年7月 – 2025年12月
- **测试数据：** 2026年1月 – 2026年3月

## 3. 特征工程

### 3.1 特征分层设计

**第一层：基础价格特征**

- OHLCV 原始数据
- 收益率（对数收益率）
- 价格变化量与变化率

**第二层：技术分析指标**

| 指标 | 类别 | 说明 |
|------|------|------|
| RSI (Relative Strength Index) | 动量指标 | 衡量超买/超卖状态 |
| MACD (Moving Average Convergence Divergence) | 趋势指标 | 捕捉趋势反转信号 |
| Bollinger Bands | 波动率指标 | 衡量价格波动范围 |
| EMA (Exponential Moving Average) | 均线指标 | 7日、25日、99日多周期均线 |
| ATR (Average True Range) | 波动率指标 | 衡量市场波动幅度 |

**第三层：高级特征（时间允许的情况下）**

- 链上数据特征：活跃地址数、交易所净流入、大额转账次数
- 舆情特征：Twitter 情感得分（正面/负面/中性比例）
- 时间特征：星期几、小时、是否为月度结算日等周期性编码

### 3.2 数据预处理

- 缺失值处理：前向填充 (forward fill)
- 归一化：MinMaxScaler 或 StandardScaler（按训练集统计量）
- 序列化：滑动窗口方式构建输入序列

## 4. 模型设计

### 4.1 价格预测模型（主线）

#### 4.1.1 预测任务定义

| 任务类型 | 定义 | 评价指标 |
|----------|------|----------|
| 回归预测 | 预测未来某一时刻的具体价格值 | MAE, RMSE, MAPE |
| 分类预测 | 预测未来价格涨跌方向 | Accuracy, F1-Score, AUC |
| 方向准确率 | 预测涨跌方向的正确比例 | Directional Accuracy |

#### 4.1.2 预测时间窗口

采用多时间尺度对比设计：

| 时间尺度 | 预测目标 | 适用场景 |
|----------|----------|----------|
| 短期 | 未来 1 小时 / 4 小时 | 日内交易参考 |
| 中期 | 未来 1 天 / 7 天 | 波段交易参考 |

#### 4.1.3 模型对比体系

**Baseline 层：传统统计与机器学习方法**

| 模型 | 用途 | 说明 |
|------|------|------|
| ARIMA | 统计基准 | 时间序列预测经典方法，作为最低基准线 |
| XGBoost | 机器学习基准 | 基于特征工程的梯度提升树，验证传统ML效果 |

**核心层：经典深度学习方法**

| 模型 | 结构 | 关键超参数 |
|------|------|------------|
| LSTM | 2–3 层，隐藏层维度 64/128 | Dropout: 0.2–0.3, BatchNorm |
| GRU | 2–3 层，隐藏层维度 64/128 | 参数更少，训练更快，与 LSTM 对比 |

**进阶层：Transformer 类方法**

| 模型 | 亮点 | 说明 |
|------|------|------|
| Transformer | 自注意力机制 | 捕捉长距离依赖和周期性模式 |
| Temporal Fusion Transformer (TFT) | 可解释性 | 注意力权重可视化，分析特征重要性 |

#### 4.1.4 训练策略

**数据划分：** 严格按时间顺序，采用滚动窗口验证（Walk-Forward Validation）。

```
时间轴 →
|--- 训练窗口 ---|-- 验证 --|-- 测试 --|
         |--- 训练窗口（滚动）---|-- 验证 --|-- 测试 --|
```

**输入序列长度（Lookback Window）：** 实验对比 24h、48h、168h（一周）三种设置。

**学习率策略：** ReduceLROnPlateau 或 CosineAnnealing，应对加密货币数据的高波动性。

**早停机制：** 基于验证集损失，patience 设为 10–15 个 epoch。

#### 4.1.5 重要实验设计：市场状态分析

将测试集按市场状态划分为三个阶段：

| 市场状态 | 定义 | 分析目标 |
|----------|------|----------|
| 牛市（上升趋势） | 连续 N 日均线向上 | 趋势捕捉能力 |
| 熊市（下降趋势） | 连续 N 日均线向下 | 风险预警能力 |
| 震荡市 | 价格在一定范围内波动 | 噪音过滤能力 |

分别评估各模型在不同市场状态下的表现，得出差异化结论。

### 4.2 异常检测模型（辅线）

**方法：** LSTM-Autoencoder

**原理：** 用正常市场数据训练 Autoencoder，使其学习"正常"市场行为的低维表示。当输入数据的重构误差超过阈值（如 3 个标准差）时，标记为异常。

**检测目标：**

- 闪崩事件（价格在短时间内剧烈下跌）
- 异常交易量突增（可能的市场操纵信号）
- 价格操纵模式（pump-and-dump）

**与价格预测的联动分析：** 对比异常事件发生前后，价格预测模型的预测精度变化情况，分析异常事件对模型可靠性的影响。

## 5. 系统设计

### 5.1 技术栈

| 组件 | 技术选择 |
|------|----------|
| 编程语言 | Python 3.10+ |
| 深度学习框架 | PyTorch |
| 数据获取 | ccxt, Binance API |
| 技术指标计算 | TA-Lib / pandas-ta |
| Web 框架 | Streamlit |
| 数据可视化 | Plotly, Matplotlib |
| 版本管理 | Git + GitHub |

### 5.2 Dashboard 功能模块

**模块一：实时数据展示**
- 接入 API 自动拉取最新价格和交易量
- K 线图 + 技术指标叠加显示
- 支持用户选择不同时间粒度（1h / 4h / 1d）

**模块二：价格预测**
- 用户选择币种和预测时间范围
- 调用训练好的模型输出预测结果
- 显示预测值、置信区间、涨跌概率

**模块三：异常警报**
- 实时监控市场状态
- 异常检测触发时弹出预警提示
- 展示历史异常事件时间线

**模块四：模型对比面板**
- 展示各模型在不同时间段的预测准确率
- 支持用户切换不同模型查看预测结果
- 注意力权重可视化（TFT 模型）

## 9. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| API 数据获取受限 | 无法获取足够的训练数据 | 提前下载历史数据备份；准备多个备选 API |
| Transformer 模型实现复杂度高 | 开发进度延迟 | 使用 PyTorch Forecasting 等封装库；设定明确的截止时间，必要时降低模型复杂度 |
| 模型预测效果不理想 | 系统实用性降低 | 对比分析本身就是研究贡献；在报告中讨论失败原因同样有价值 |
| 团队成员协作问题 | 进度不一致 | 每周例会 + 使用 GitHub 进行代码管理和任务追踪 |
